参考

  • GCC: GNU Compiler Collection, GCC是一些主流编程语言的编译器的合集。也可以理解为GNU C Compiler。
  • G++: c++的编译器。
    • -std c++11 开启c++11支持
    • Options for Directory Search
      • 参考自 https://linux.die.net/man/1/g++ Options for Directory Search
      • -Idir （大写i）: 将dir添加到header files的搜索路径中，dir的搜索优先级比系统搜索路径优先级高
      • -Ldir (大写L) ：将dir添加到 -l (小写L)的搜索路径中。
    • Options Controlling the Preprocessor
      • -I dir (大写i) : 将dir添加到header files的搜索路径中，dir的搜索优先级比系统搜索路径优先级高
    • Options for Linking
      • -llibrary / -l library （小写L） : 推荐使用第一种，第二种分开的格式只是为了支持POSIX；在link的过程中，搜索名为liblibrary.a的可链接文件（官方文档中的描述）。一些中文博客中说，link时先找动态链接，即先搜索liblibrary.so文件，找不到so才去找liblibrary.a（真实性存疑）。
  • Clang: From Xcode 4.2, Clang is the default compiler for Mac OS X.

Reserved Words

static

作用：控制变量的存储方式和可见性。

  • static变量存储在静态存储区，不在栈上。（存储方式）

  • 全局static变量在程序开始运行时进行初始化，程序结束运行时收回，所以生命周期变长了。（局部变量是在第一次运行时初始化）

  • static修饰一个全局变量/函数，则这个全局变量/函数只能在本文件中被访问。即使extern外部声明也不行。

    • 全局变量和static全局变量的区别：全局变量默认是有外部链接性的，作用域是整个工程，可使用extern。

在class中的作用

static的使用场景：需要一个变量/函数为整个类服务，而非某个对象，同时又不破坏类的封装性，即该变量/函数要在类的内部。

static成员变量

  • 无论创建多少个对象，静态成员变量只有一个副本，此副本在类的所有对象中是共享的。
  • 如果不存在初始化语句，则在创建第一个对象时，初始化为0。
  • 初始化不能放在类的定义中。

static成员函数

  • 静态成员函数与类的特定对象无关，不存在对象时也能被调用
  • 静态成员函数只能访问类的静态成员变量、其他静态成员函数和类外部的函数

void

  • void字面意思是“无类型”，void * 即无类型的指针，void * 可以指向任意类型的数据。

  • void的作用：

    • 对函数返回的限定：函数不返回数据时，应该注明返回void类型。

    • 对函数参数的限定：如果函数的参数可以是任意类型指针，那么应声明其参数为void *
